  1. A Programmer's Quick and Dirty History of the Web

    Erich Stehr

  2. Who am I? What are we doing here?

    • Professional software developer for 34 years

    • Web development off and on for 21 years

    • Goal is not to teach you to be web developers in 30 minutes
    • Introduction to some of the concepts, sites, and search queries you will need to research in order to consider developing a web application.
    • Assumptions from my previous CodeDay: you are learning Java, but have not finished the book and have no web development experience.

  3. SGML (1969, 1986)

    • At IBM, the Generalized Markup Language was created in 1969, and ISO standardized in 1986
    • Creates markup languages, marking up document content elements with tags and attributes
    • <!DOCTYPE html>
<html>
    <head>
        <title>Page title</title>
    </head>
    <body>
        <p class="greeting">Hello, World</p>
        <img src="http://placehold.it/320x100" />
    </body>
</html>

  4. HTML (1993)

    • Tim Berners-Lee at CERN, using SGML to create HyperText Markup Language
    • Goal: publishing physics papers over the Internet
    • block vs. inline
    • <form> and <input/> tags: passing data to server
    • later: tables
      1Internet draft
      2.0RFC 1866,1867,1942,1980,2070
      3.2W3C recommendation

  5. CSS (1996): content vs. presentation

    • <blink>, <marquee> tags, among others
    • selectors: E#id.class
    • specificity:
      • !important
      • inline
      • ID count
      • attribute/class count
      • element count
    • positioning: flow, float, absolute|relative|fixed
    • //csszengarden.com (2003) currently 218 examples of what the same HTML and different CSS can look like

  6. W3C drifts off into the (data) weeds

    • XML (1997): 80% of SGML's power with 20% of the complexity
    • XHTML (2000): HTML under XML's more restrictive rules
    • Semantic Web (or Web of Data)
      • RDF (Resource Data Framework)
      • OWL (Web Ontology Language)
      • LOD (Linked Open Data)
    • https://github.com/JoshData/rdfabout/blob/gh-pages/intro-to-rdf.md

  7. WHAT-WG takes over, HTML5 (2005, 2007)

    • Web Hypertext Application Technology Working Group
    • Started with a second version of Forms; W3C refused, then gave in to ultimatum
    • Bent in a different direction
      • No longer SGML based
      • Focus on applications, not documents

  8. HTTP

    • HyperText Transfer Protocol
      • Stateless
      • request/response
      • status codes (200 OK, 404 Not Found)
    • Roy Fielding, REST
      • Representational State Transfer
      • Resources (URIs)
      • Verbs
        • GET
        • PUT, POST
        • DELETE
        • ...
      • Links to next allowable states (Hypertext As The Engine Of Application State; HATEOAS)

  9. JavaScript, ECMAScript (1995, 1997)

    • template-based OO, not class-based
    • imperative/functional/scripting
    • asynchronous but not concurrent
    • XmlHttpRequest object (XHR, 1999), AJAX (2005)
    • JavaScript Object Notation (JSON, 2002, RFC 7159) vs. XML

  10. Frameworks

      • jQuery, QUnit
      • Angular vs. Ember vs. Backbone
      • React
      • Bootstrap
      • Bower
    • Near-requirements for current web development
    • learning curve and frameworks

  11. Guidance

    • Data zones and data transfers
      • Client-side
      • Server-side
      • Transfer up
      • Transfer down
    • Pick up jQuery and Angular

  12. Use cases

    • Chat (forms, websockets, SignalR/SignalJ/Spring 4)
    • Turn-based games (add websockets for less turn based operations)
    • Data requests
    • Presentations

  13. Conclusion

    • The trinity of HTML, HTTP, and JavaScript
    • Polyglot programming
    • Use the frameworks, but know what they do
